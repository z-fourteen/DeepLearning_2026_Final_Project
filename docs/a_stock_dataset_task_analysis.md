# A股数据集与深度学习任务技术分析文档

本文档从“深度学习项目导师”的视角，解释当前项目中的 A 股数据、作业任务目标，以及后续进行数据预处理、特征工程、模型设计和回测时应注意的问题。

核心结论：本项目不是单只股票预测，而是一个“多股票、日频、面板时间序列预测 + 排序交易”的任务。数据已经比较完整，难点不在“有没有数据”，而在“如何严格按时间构造样本，避免未来信息泄漏”。

## 1. 数据集整体认知

A 股数据可以理解为中国股票市场中每只上市公司股票每天的交易记录。

### 1.1 股票代码

股票代码 `ts_code` 是股票的唯一身份标识，例如：

- `000001.SZ`：深交所股票。
- `600000.SH`：上交所股票。

后缀含义：

- `.SZ`：深圳证券交易所。
- `.SH`：上海证券交易所。

作业要求正式模拟交易范围为“除 ST 股、北交所股票之外的所有 A 股上市股票”，因此后续需要结合 `basic.csv` 和 `stock_st/` 进行股票池过滤。

### 1.2 K线/日线数据

K 线/日线数据就是把一天的交易压缩成一行。典型字段如下：

| 字段        | 含义                 | 对模型的意义                       |
| ----------- | -------------------- | ---------------------------------- |
| `open`      | 开盘价               | 当天交易开始时市场定价             |
| `high`      | 最高价               | 当日上冲幅度，反映波动             |
| `low`       | 最低价               | 当日下探幅度，反映风险             |
| `close`     | 收盘价               | 最常用的价格基准，也是标签常用来源 |
| `pre_close` | 昨收价/除权昨收价    | 用于计算涨跌额、涨跌幅             |
| `change`    | 涨跌额               | 通常为 `close - pre_close`         |
| `pct_chg`   | 涨跌幅百分比         | 比绝对价格更适合跨股票建模         |
| `vol`       | 成交量，单位通常为手 | 反映交易活跃度                     |
| `amount`    | 成交额               | 反映资金规模                       |
| `vwap`      | 成交量加权平均价     | 比收盘价更接近全天平均成交价格     |

对深度学习而言，原始价格本身不太适合直接输入，因为不同股票价格量级差异很大。更常用的是收益率、涨跌幅、成交量变化率、换手率、波动率、技术指标等相对特征。

### 1.3 复权数据

复权是金融数据中非常重要的概念。股票分红、送股、拆股会让价格出现机械跳变，如果不复权，模型可能误以为发生了巨大涨跌。

当前数据中没有看到单独的 `adj_factor` 字段，但 README 中提到 `pre_close` 和 `pct_chg` 基于除权昨收计算。因此：

- 标签优先使用 `pct_chg` 或基于连续口径构造的收益率。
- 如果后续发现价格序列因除权断裂明显，需要额外寻找复权因子。

### 1.4 时间序列数据

股票数据是典型时间序列数据。关键不是“有多少行”，而是“顺序”。

股票预测中绝对不能随机打乱时间后再划分训练集和验证集，否则未来信息会泄漏到过去。例如，用 2025 年数据参与标准化后再评估 2024 年样本，就是隐含使用了未来信息。

## 3. 核心数据结构分析

### 3.1 `daily/`
这些是金融时间序列预测中最基础、也是最重要的**行情数据（Market Bar Data）**。它们记录了每一个交易周期（如日线、分钟线）最直接的价格和成交量变动。

---

#### 1. 基础标识与核心价位

这些构成了每个交易周期的“四大价位（OHLC）”，是构建所有技术指标的基石。

* **`ts_code` / `trade_date`：**
* **概念：** 股票唯一代码与交易日期（参考前文解释）。
* ⚠️ **DL 处理建议：** 必须从特征矩阵中剔除，仅作为主键或划分数据集的依据。


* **`open` (Opening Price / 开盘价)：**
* **概念：** 交易日开盘时的第一笔成交价格。


* **`high` (Highest Price / 最高价)：**
* **概念：** 交易日内成交的最高价格。反映了多头（买方）当天能推高的极限。


* **`low` (Lowest Price / 最低价)：**
* **概念：** 交易日内成交的最低价格。反映了空头（卖方）当天能打压的极限。


* **`close` (Closing Price / 收盘价)：**
* **概念：** 交易日结束时的最后一笔成交价格。是全天市场共识的最终体现，通常也是时序预测的**核心目标（Target）**。


* **`pre_close` (Previous Closing Price / 昨收盘价)：**
* **概念：** 前一个交易日的收盘价（注意：在有分红送股的情况下，通常需要使用**复权后的昨收盘价**，否则价格跳空会导致模型误判）。



---

#### 2. 涨跌幅与变化量

反映了市场的动量（Momentum）和波动剧烈程度。

* **`change` (Price Change / 涨跌额)：**
* **概念：** 当天收盘价相对于前一日收盘价的绝对差值。计算公式为：$\text{close} - \text{pre\_close}$。
* ⚠️ **DL 处理建议：** 由于不同股票的价格基数不同（2元的股票和200元的股票，同样涨跌1元概念完全不同），**不建议直接作为通用模型的特征**。


* **`pct_chg` (Percentage Change / 涨跌幅)：**
* **概念：** 当天收盘价相对于前一日收盘价的涨跌百分比。计算公式为：$(\text{close} - \text{pre\_close}) \div \text{pre\_close} \times 100\%$。
* 💡 **特征意义：** 这是一个**天然平稳化**的优良特征，消除了价格绝对值的影响。非常适合直接作为神经网络的输入，用来捕获股票的短期动量。



---

#### 3. 成交量与资金流

价格需要成交量的配合才真实。神经网络通过“量价结合”能更有效地捕捉主力资金的意图。

* **`vol` (Volume / 成交量)：**
* **概念：** 交易日内该股票成交的股票总股数（国内 A 股通常以“手”为单位，1手 = 100股）。


* **`amount` (Amount / 成交额)：**
* **概念：** 交易日内该股票成交的总金额（单位通常为人民币元或千元）。计算公式为：$\sum (\text{每笔成交价} \times \text{该笔成交股数})$。
* ⚠️ **DL 处理建议：** `vol` 和 `amount` 同样存在严重的股票个体差异（大盘股成交几十亿，小盘股成交几百万）。在喂给模型前，**必须进行横截面标准化（Cross-sectional Standardization）**，或者转化为“换手率”、“成交量变动率”等相对指标。


* **`vwap` (Volume Weighted Average Price / 成交量加权平均价)：**
* **概念：** 按照全天每笔交易的成交量作为权重，计算出的日内平均价格。计算公式为：$\text{amount} \div \text{vol}$。
* 💡 **大作业亮点：** 机构投资者（如公募、私募）在建仓时，通常以 `vwap` 作为衡量交易成本的标准。相比于容易在尾盘被操纵的 `close`（收盘价），`vwap` 更能反映全天资金真实的平均成本。在做高频预测或多因子挖掘时，**将 `vwap` 作为特征往往能带来比 `close` 更好的效果**。



---

#### 🛠️ 大作业加分项：衍生量价特征工程 (Feature Derivatives)

如果只是把原始的 OHLC 喂给模型，LSTM 或 Transformer 很难直接学到高阶规律。你可以在报告中写道，你基于原始行情字段手工构建了以下**衍生特征**：

1. **日内振幅（Intraday Amplitude）：**
反映当天的波动率（Volatility），可以直接输入神经网络：

$$\text{Amplitude} = \frac{\text{high} - \text{low}}{\text{pre\_close}}$$


2. **K线实体比例（Body Ratio）：**
反映多空博弈的最终胜负强弱：

$$\text{Body\_Ratio} = \frac{\text{close} - \text{open}}{\text{high} - \text{low}}$$


3. **收盘价相对位置（Close Location）：**
看收盘价是接近最高点（强）还是最低点（弱）：

$$\text{CLS\_Loc} = \frac{\text{close} - \text{low}}{\text{high} - \text{low}}$$

`daily` 是最核心的数据。以 `daily/20260522.csv` 为例，有 5504 行，每行是一只股票当天的数据，没有缺失值。

字段为：

```text
ts_code, trade_date, open, high, low, close, pre_close,
change, pct_chg, vol, amount, vwap
```

它是“日期切片格式”，不是模型最喜欢的“单股票连续序列格式”。如果要训练 LSTM/Transformer，需要先按 `ts_code` 把多个日期文件拼接起来，再按时间排序，变成：

```text
股票 A: t1, t2, t3, ...
股票 B: t1, t2, t3, ...
...
```

### 3.2 `metric/`

这些概念是你在金融时间序列预测中（尤其是使用国内 A 股数据时）会遇到的最核心的**量化特征（Features）**。

在深度学习模型中，它们不能被一股脑地直接喂进去。为了方便你写大作业的**数据预处理**和**特征工程**部分，我将这些概念分为五大类进行详细拆解，并附带了**深度学习处理建议**：

---

#### 1. 基础标识与时间维度

这两列是数据的“主键”，用来对齐时间序列，**绝对不能**作为特征输入模型。

* **`ts_code` (Tushar Stock Code / 股票代码)：**
* **概念：** 股票在交易所的唯一身份代码。例如 `000001.SZ`（平安银行）。
* ⚠️ **DL 处理建议：** 模型训练时必须剔除。如果是多股票共同训练，通常会将其转化为类别变量进行 **Embedding（嵌入）** 处理。


* **`trade_date` (Trading Date / 交易日)：**
* **概念：** 该笔数据发生的公开交易日期，格式通常为 `YYYYMMDD`。
* ⚠️ **DL 处理建议：** 必须剔除。但它是你划分训练集、验证集和测试集（Time Series Split）的唯一依据。



---

#### 2. 价格与交投活跃度 (Price & Liquidity)

金融时序预测的灵魂。神经网络对这些指标的变动非常敏感。

* **`close` (Closing Price / 收盘价)：**
* **概念：** 股票在当天交易结束时的最后一笔成交价格。
* ⚠️ **DL 处理建议：** **千万不要直接喂绝对价格！** 因为价格序列是非平稳的（有趋势）。你必须将其转化为**对数收益率（Log Return）**：$\log(Price_t / Price_{t-1})$，或者进行归一化。


* **`turnover_rate` (Turnover Rate / 换手率)：**
* **概念：** 当天股票的成交股数占**总股本**的比例。反映了股票的流动性和市场关注度。


* **`turnover_rate_f` (Free Float Turnover Rate / 自由流通股换手率)：**
* **概念：** 当天成交股数占**自由流通股本**（剔除了大股东持股、锁定期股票等实际不参与交易的股本）的比例。
* 💡 **大作业亮点：** 在 A 股市场，它比普通的换手率更能真实反映散户和游资的实际交投活跃度。深度学习模型输入时，**优先选择这一列**。


* **`volume_ratio` (Volume Ratio / 量比)：**
* **概念：** 用来衡量当天成交量是否异常放大的指标。计算公式为：$\text{当天开盘后平均每分钟成交量} \div \text{过去5个交易日平均每分钟成交量}$。
* 💡 **特征意义：** 如果量比大于 1（比如大于 2 或 3），说明当天有突发利好/利空，资金在疯狂涌入或出逃，是动量模型（Momentum）非常喜欢的特征。



---

#### 3. 估值指标 (Valuation Ratios)

这类指标反映了股票是“便宜”还是“贵”，在传统多因子模型中被称为**价值因子（Value Factors）**。

* **`pe` (Price-to-Earnings Ratio / 市盈率-静态)：**
* **概念：** $\text{每股价格} \div \text{上年度每股收益(EPS)}$。


* **`pe_ttm` (Trailing Twelve Months PE / 市盈率-滚动)：**
* **概念：** $\text{每股价格} \div \text{最近4个季度每股收益的总和}$。
* 💡 **应用建议：** 它比静态 `pe` 更具时效性，因为它用的是最新的季度财报。时序预测中**优先使用 `pe_ttm**`。


* **`pb` (Price-to-Book Ratio / 市净率)：**
* **概念：** $\text{每股价格} \div \text{每股净资产}$。通常用于评估银行、地产等重资产行业。


* **`ps` (Price-to-Sales Ratio / 市销率-静态)：**
* **概念：** $\text{每股价格} \div \text{每股销售收入}$。


* **`ps_ttm` (Trailing Twelve Months PS / 市销率-滚动)：**
* **概念：** $\text{每股价格} \div \text{最近4个季度每股销售收入总和}$。常用于评估尚未盈利的高科技企业或初创公司。



---

#### 4. 股息与分红 (Dividend Ratios)

* **`dv_ratio` (Dividend Yield / 股息率-静态)：**
* **概念：** $\text{过去一年每股总分红} \div \text{当前股价} \times 100\%$。反映了买入这只股票拿分红的收益率。


* **`dv_ttm` (Trailing Twelve Months Dividend Yield / 股息率-滚动)：**
* **概念：** 采用最近12个月内已经实施的分红数据计算的股息率。在红利资产大火的背景下，这是一个非常强的防御性特征。



---

#### 5. 股本与市值 (Share Capital & Market Value)

这类指标反映了公司的“体量”大小（常用于捕捉**小市值效应**）。

* **`total_share` (Total Shares / 总股本)：** 公司发行的总股票数量。
* **`float_share` (Float Shares / 流通股本)：** 可以在二级市场公开交易的股票数量。
* **`free_share` (Free Float Shares / 自由流通股本)：** 剔除持股 5% 以上非流通大股东等之后的、市场中实际每天在换手的最活跃股本。
* **`total_mv` (Total Market Value / 总市值)：** $\text{总股本} \times \text{当前股价}$。代表公司的总价值。
* **`circ_mv` (Circulating Market Value / 流通市值)：** $\text{流通股本} \times \text{当前股价}$。代表市场中实际在交易的股票价值。

---

#### 🛠️ 深度学习大作业预处理核心公式

你在写报告的“数据预处理”章节时，可以用上以下两个标准操作，可以作为你报告的加分项：

1. **特征平稳化（Log-Transform & Standardize）：**
像 `total_mv`（总市值）这样的数据，大公司和小公司能差几个数量级。直接喂进去会导致神经网络梯度爆炸。你可以先取对数，再做标准化：

$$X_{\text{scaled}} = \frac{\log(total\_mv) - \mu}{\sigma}$$


2. **处理极值（Winsorization / 去极值）：**
金融数据（如 `pe_ttm`）经常会有极大的异常值（比如某公司亏损导致 PE 变成几千倍）。在喂给模型之前，一定要进行**截断处理**（比如把大于 99% 分位数和小于 1% 分位数的值强制拉回边界），否则一个异常值就会毁掉整个 Batch 的训练效果。

`metric` 与 `daily` 可以通过 `ts_code + trade_date` 合并。以 2026-05-22 为例，`metric` 也是 5504 行，但存在缺失值：

- `pe` 缺失 1455 个。
- `pe_ttm` 缺失 1543 个。
- `dv_ratio` 缺失 1742 个。
- `dv_ttm` 缺失 1847 个。

这是正常现象。亏损公司 PE 为空，部分股息率也可能为空。基本面特征不能简单全部丢弃，否则会损失大量股票；建议用缺失指示变量 + 中位数/截面填充。

### 3.3 `moneyflow/`

`moneyflow` 以 2026-05-22 为例有 5191 行，少于 `daily` 的 5504 行，说明不是所有股票都有资金流数据。

它适合作为增强特征，但合并后需要处理缺失。常见做法：

- 资金流金额类特征除以 `amount` 或市值，变成相对资金流。
- 缺失值可以填 0，同时增加 `has_moneyflow` 指示变量。


这些是以 Tushare 或 AkShare 为代表的国内金融数据接口中，最核心的**资金流向（Money Flow）数据**，通常被称为**个股资金大单/主力资金监测数据**。

在金融深度学习中，这组数据非常珍贵。因为它泄露了日内交易中**不同体量资金的博弈掀背战**。神经网络（尤其是 Transformer 或 LSTM）对这类结构化特征极其敏感，能有效识别出“机构建仓”或“散户接盘”的信号。

为了方便你写大作业，我按照资金体量和最终指标将其归纳为五大类进行拆解，并附带了**深度学习特征工程的独家建议**：

---

#### 1. 散户与游资资金 (Small & Medium)

金融学里通常称这部分资金为“噪声交易者（Noise Traders）”。

* **`buy_sm_vol` / `buy_sm_amount` (Buy Small Volume/Amount)：** 小单买入量（股）/ 小单买入金额（元）。
* **`sell_sm_vol` / `sell_sm_amount` (Sell Small Volume/Amount)：** 小单卖出量 / 小单卖出金额。
* 💡 **概念：** 对应单笔成交金额极小的单子（如 A 股中单笔小于 4 万元）。通常代表散户（Retail Investors）的散碎买卖行为。


* **`buy_md_vol` / `buy_md_amount` (Buy Medium Volume/Amount)：** 中单买入量 / 中单买入金额。
* **`sell_md_vol` / `sell_md_amount` (Sell Medium Volume/Amount)：** 中单卖出量 / 中单卖出金额。
* 💡 **概念：** 对应中等金额的成交（如单笔 4 万–20 万元）。通常代表大散户或游资（Hot Money）的试探性行为。



---

#### 2. 机构与主力资金 (Large & Extra-Large)

量化投资和深度学习模型**最需要盯紧的特征**。在金融学里被称为“知情交易者（Informed Traders）”。

* **`buy_lg_vol` / `buy_lg_amount` (Buy Large Volume/Amount)：** 大单买入量 / 大单买入金额。
* **`sell_lg_vol` / `sell_lg_amount` (Sell Large Volume/Amount)：** 大单卖出量 / 大单卖出金额。
* 💡 **概念：** 单笔成交金额较大（如单笔 20 万–100 万元）。通常是**地方游资、普通私募或大户**在调仓。


* **`buy_elg_vol` / `buy_elg_amount` (Buy Extra-Large Volume/Amount)：** 特大单买入量 / 特大单买入金额。
* **`sell_elg_vol` / `sell_elg_amount` (Sell Extra-Large Volume/Amount)：** 特大单卖出量 / 特大单卖出金额。
* 💡 **概念：** 单笔成交金额极大（如单笔大于 100 万元）。这几乎 100% 是**公募基金、国家队、外资（北向资金）或绝对主力机构**在进行大举进攻或撤退。



---

#### 3. 资金轧差与净流入 (Net Money Flow)

直接总结当天主力资金态度的一手指标。

* **`net_mf_vol` (Net Money Flow Volume / 净流入量)：**
* **概念：** 交易日内主力资金的净买入股数。


* **`net_mf_amount` (Net Money Flow Amount / 净流入额)：**
* **概念：** 交易日内主力资金的净买入金额。
* 📊 **计算公式（底层逻辑）：**

$$\text{net\_mf\_amount} = (\text{buy\_lg\_amount} + \text{buy\_elg\_amount}) - (\text{sell\_lg\_amount} + \text{sell\_elg\_amount})$$


* 💡 **大作业亮点：** 这个指标大于 0，说明机构在净买入，股价有推升动力（主力吸筹）；小于 0 说明机构在撤退（主力派发）。



---

#### 🧠 深度学习大作业进阶：如何挖掘这组数据的神级特征？

如果你只是把这些原始的 `amount` 或 `vol` 直接堆给 LSTM，模型大概率会因为规模无法对齐而崩溃。在写大作业的**特征工程（Feature Engineering）**章节时，强烈建议你手动计算并引入以下**衍生指标**，老师一定会对你的金融洞察力刮目相看：

##### 1. 机构散户博弈度 (Institutional-Retail Ratio)

直接衡量当天是机构在主导行情，还是散户在盲目跟风。


$$\text{IR\_Ratio} = \frac{\text{buy\_elg\_amount} + \text{buy\_lg\_amount}}{\text{buy\_sm\_amount} + \text{buy\_md\_amount}}$$

* **DL 价值：** 如果股价上涨且 `IR_Ratio` 极高，说明是健康的机构推升；如果股价上涨但 `IR_Ratio` 极低，说明是散户接盘的情绪博弈，短期内极易见顶回落。

##### 2. 散户接盘率 (Retail Trapping Rate)

用来捕捉经典的“主力拉高出货，散户高位接盘”形态。


$$\text{Retail\_Trap} = \frac{\text{buy\_sm\_amount} - \text{sell\_sm\_amount}}{\text{amount}}$$

* **DL 价值：** 这个相对比例天然处于 $[-1, 1]$ 之间，非常适合神经网络直接吸收。当该指标连续数天为正且处于高位时，序列模型（如 GRU）能敏锐地捕捉到见顶的先兆信号。

##### 3. 净流入强度 (Net Flow Intensity)

消除股票自身市值的绝对规模影响，让不同股票具有横向可比性。


$$\text{Net\_Intensity} = \frac{\text{net\_mf\_amount}}{\text{amount}}$$

* **DL 价值：** 算出了主力净流入占全天总成交额的比例。这是一个极其稳健的多因子输入特征（价值动量），能极大缓解模型深度增加时的过拟合风险。

### 3.4 `basic.csv`

`basic.csv` 是静态股票信息。重要字段包括：

| 字段        | 用途                                         |
| ----------- | -------------------------------------------- |
| `market`    | 过滤北交所；区分主板、创业板、科创板         |
| `industry`  | 可做行业 one-hot/embedding，也可做行业中性化 |
| `list_date` | 过滤上市时间太短的新股                       |
| `name`      | 可辅助识别 ST，但更可靠的是 `stock_st/`      |

统计结果显示当前共有 5522 只股票，其中：

| 市场   | 数量 |
| ------ | ---: |
| 主板   | 3200 |
| 创业板 | 1398 |
| 科创板 |  609 |
| 北交所 |  315 |

正式股票池建议先过滤 `market == "北交所"`。

### 3.5 `trade_cal.csv`

`trade_cal.csv` 用来定义“连续交易日”，不能用自然日直接滑窗，因为周末和节假日没有交易。

核心字段：

| 字段            | 含义                           |
| --------------- | ------------------------------ |
| `exchange`      | 交易所                         |
| `cal_date`      | 日期                           |
| `is_open`       | 是否开市，1 为交易日，0 为休市 |
| `pretrade_date` | 上一交易日                     |

### 3.6 `market/`

`market/000300.SH.csv` 等指数文件是按单指数存储的完整时间序列，日期是倒序。

它们适合：

- 构造市场环境特征，例如沪深300当日涨跌幅、过去5日市场收益率。
- 作为回测 benchmark，例如和沪深300、上证指数对比。

### 3.7 `stock_st/`

`stock_st/` 提供每日 ST 股票列表。作业要求排除 ST 股，因此训练和模拟交易时都应该按日期过滤。

注意不要用未来 ST 状态过滤过去样本。正确方式是：在日期 `t` 的样本中，只使用 `t` 当天或此前可知的 ST 状态。

### 3.8 `news/`

`news/` 是文本数据，字段包括：

```text
datetime, content, title
```

新闻数据从 2019 年开始，处理难度较高。第一版模型不建议把新闻作为主线，因为它需要：

- 文本编码。
- 新闻与股票的关联抽取。
- 新闻发布时间和交易时间对齐。
- 防止盘后新闻误用于盘中交易。

它更适合作为加分项或第二阶段扩展。

## 4. 作业任务目标分析

PDF 明确说明：本次作业是“基于深度学习的股票趋势预测与模拟交易”。

本质任务是：

```text
用过去一段时间的股票特征序列
-> 预测未来短期收益方向或收益分数
-> 根据分数排序买卖股票
```

它既可以做分类，也可以做回归：

| 任务定义 | 标签                             | 优点                    | 缺点                    |
| -------- | -------------------------------- | ----------------------- | ----------------------- |
| 二分类   | 未来 n 日收益率 > 0 为 1，否则 0 | 简单，方向胜率好解释    | 忽略收益幅度            |
| 三分类   | 涨/平/跌                         | 对噪声更稳健            | 阈值需要设计            |
| 回归     | 未来 n 日收益率                  | 可直接用于排序、IC 评估 | 噪声大，loss 不一定好看 |
| 排序学习 | 预测股票相对排名                 | 和交易目标更一致        | 实现复杂                |

结合 PDF 要求里的 IC、ICIR、交易打分，建议优先做“回归预测未来收益率”，模型输出一个连续分数。交易时不一定相信分数的绝对值，只用它在股票之间排序。

### 4.1 样本构造方式

一个合理样本可以定义为：

```text
输入 X_i,t = 股票 i 在 t-L+1 到 t 日的特征序列
标签 y_i,t = 股票 i 在 t+1 到 t+h 的未来收益率
```

例如：

```text
窗口长度 L = 20
预测周期 h = 1

X = 过去20个交易日的 pct_chg、vol变化、amount变化、技术指标等
y = 下一交易日收益率
```

深度学习数据形状可以设计为：

```text
X: [num_samples, window_length, num_features]
y: [num_samples]
```

### 4.2 时间点约束

由于数据是盘后更新，如果在 2026-04-21 交易，最多只能使用 2026-04-20 及以前的数据。

也就是说：

- 不能用当天 `open/high/low/close` 去指导当天买入。
- 如果样本输入窗口截止到 `t`，交易动作最早发生在 `t+1`。
- 标签应对应 `t+1` 或更远未来的收益。

## 5. 数据预处理建议

最重要的预处理原则：所有处理都要模拟真实交易时的信息边界。

推荐流程：

1. 读取 `trade_cal.csv`，得到有序交易日。
2. 逐日读取 `daily`，合并 `metric`、`moneyflow`。
3. 用 `basic.csv` 过滤北交所股票。
4. 用 `stock_st/YYYYMMDD.csv` 过滤当天 ST 股票。
5. 过滤上市时间太短的股票，例如上市不足 120 个交易日不参与训练。
6. 构造收益率、成交量变化率、技术指标。
7. 按时间划分训练/验证/测试，禁止随机划分。
8. 构造 sliding window。
9. 只用训练期统计量做标准化，验证/测试复用训练期参数。

### 5.1 字段处理建议

| 字段类型                          | 建议处理                                     |
| --------------------------------- | -------------------------------------------- |
| 价格类 `open/high/low/close/vwap` | 不直接用绝对值，改成相对收益或相对收盘价比例 |
| `pct_chg`                         | 可直接作为核心特征，但注意只能用历史日       |
| `vol/amount`                      | 建议 `log1p` 后再标准化                      |
| `turnover_rate/volume_ratio`      | 可标准化，异常值 winsorize                   |
| `pe/pb/ps`                        | 缺失多，建议截面中位数填充 + 缺失指示        |
| 市值 `total_mv/circ_mv`           | `log1p`，否则大市值股票量级压制模型          |
| 资金流金额                        | 可除以 `amount` 或市值，转成相对资金流       |

### 5.2 停牌、缺失值、异常值

| 情况                   | 建议                                                     |
| ---------------------- | -------------------------------------------------------- |
| 某股票某天不在 `daily` | 视为不可交易，不强行补价格                               |
| 构造窗口时中间缺交易日 | 若是股票缺失，丢弃该窗口；若是周末节假日，用交易日历跳过 |
| `metric` 缺失          | 中位数填充 + mask                                        |
| `moneyflow` 缺失       | 可填 0 并加 `has_moneyflow` 指示                         |
| 极端涨跌/异常成交量    | 对连续特征做 1%/99% 分位裁剪                             |

### 5.3 数据集切分

建议切分方式：

```text
训练集：2019-01-01 到 2023-12-31
验证集：2024-01-01 到 2025-12-31
模拟/近未来测试：2026 年数据
```

也可以按 PDF 示例使用：

```text
训练期：2019-01 到 2024-12
验证期：2025-01 到 2025-12
```

不要把 2025 年数据混入标准化统计后再评估 2025 年，否则会发生轻微但真实的数据泄漏。

### 5.4 不同股票长度不一致

不同股票上市时间不同，停牌情况不同，序列长度天然不一致。

建议做法：

- 对每只股票单独按时间滑窗。
- 能构造出完整窗口就保留。
- 不能构造完整窗口就跳过。
- 最后把所有股票的窗口样本拼接成训练集。

不要为了统一长度，把大量缺失价格随意填充为 0，这会让模型学到虚假的价格模式。

## 6. 特征工程建议

当前已有基础特征：

```text
open, high, low, close, pre_close, change, pct_chg, vol, amount, vwap
```

### 6.1 推荐优先构造的基础特征

| 特征              | 解释                                             |
| ----------------- | ------------------------------------------------ |
| `return_1d`       | `close / pre_close - 1` 或直接用 `pct_chg / 100` |
| `intraday_return` | `close / open - 1`                               |
| `high_low_range`  | `high / low - 1`，衡量日内波动                   |
| `close_to_high`   | `close / high - 1`，收盘是否接近最高             |
| `close_to_low`    | `close / low - 1`                                |
| `log_vol`         | `log1p(vol)`                                     |
| `log_amount`      | `log1p(amount)`                                  |
| `vol_change`      | 成交量相对过去均值变化                           |
| `market_return`   | 沪深300/上证指数收益率                           |
| `relative_return` | 个股收益 - 市场收益                              |

### 6.2 技术指标

技术指标可以作为模型输入，但不要堆太多。

| 指标           | 作用                  |
| -------------- | --------------------- |
| MA5/MA10/MA20  | 趋势                  |
| Close/MA       | 当前价格相对均线位置  |
| MACD           | 中短期动量            |
| RSI            | 超买超卖              |
| Bollinger Band | 价格相对波动区间位置  |
| Volatility     | 过去 N 日收益率标准差 |
| Momentum       | 过去 N 日累计收益     |
| Turnover mean  | 流动性                |

对深度学习来说，技术指标不是越多越好。LSTM/Transformer 理论上可以从原始序列中学习均线、动量等模式；但在金融数据噪声很高、样本有限时，适量技术指标能降低学习难度。

第一版建议控制在 20-50 个数值特征，不要一开始堆到几百个。

### 6.3 可能冗余的特征

以下特征之间可能高度相关：

```text
change 和 pct_chg
open/high/low/close 的绝对值
total_mv 和 circ_mv
多组相近窗口的 MA
```

冗余特征不一定不能用，但会增加模型训练难度和过拟合风险。第一版应优先保证特征少而干净。

### 6.4 外部信息

行业和市场指数很值得用：

- `industry` 可做行业 one-hot 或 embedding。
- 指数收益可表示市场环境。
- 个股收益减去指数收益可表示超额收益。

新闻文本可以作为加分项，但处理复杂，需要文本编码、时间对齐和股票关联抽取；第一版不建议把新闻作为主线。

## 7. 数据泄漏高风险点

| 风险               | 示例                                          | 正确做法                           |
| ------------------ | --------------------------------------------- | ---------------------------------- |
| 用未来价格做特征   | 用 `t+1` 的 close 预测 `t+1` 收益             | 特征只能到 `t` 或更早              |
| 全量标准化         | 用 2019-2026 全部数据算均值方差               | 只用训练集，或滚动窗口标准化       |
| 随机划分样本       | 同一股票 2025 年样本进训练，2020 年样本进验证 | 按时间切分                         |
| 当天交易用当天收盘 | 用 4月21日收盘特征指导 4月21日买入            | 4月21日交易只能用 4月20日及以前    |
| ST 过滤用未来名单  | 用全年 ST 状态过滤过去                        | 每天只用当天或此前可知状态         |
| 指数成分股未来信息 | 用 2026 年沪深300成分训练 2019 年             | 用对应月份成分，或直接不用成分约束 |

如果历史回测结果异常好，优先怀疑数据泄漏。

## 8. 推荐第一版建模路线

建议第一版不要直接做大而全 Transformer，而是走一个稳妥闭环：

```text
股票池：非北交所、非 ST、上市满 120 交易日
数据：daily + metric + market，moneyflow 可作为第二版
标签：未来 1 日或 5 日收益率
窗口：过去 20 或 30 个交易日
模型：MLP baseline + LSTM/GRU + Transformer
评估：MSE/MAE + IC + ICIR + TopK 回测
策略：每日/每几日选预测分最高的 n 只，卖出低分 k 只
```

样本形状：

```text
window_length = 20
num_features = 30

X_i,t = 股票 i 过去20个交易日的30维特征
y_i,t = 股票 i 下一交易日收益率
```

对应张量：

```text
X: [num_samples, window_length, num_features]
y: [num_samples]
```

## 9. 回测和评价指标

PDF 中要求关注：

- 训练集/验证集 loss 变化。
- IC 值：模型预测分数与未来 n 日收益率之间的相关系数。
- ICIR：IC 均值 / IC 波动。
- 方向胜率，可选。
- 历史回测收益。
- 年化收益率。
- 夏普比率。
- 最大回撤。

建议从最简单的 TopK 策略开始：

```text
第一天等权买入预测分数最高的 n 只股票。
之后每天卖出当前持仓中得分最低的 k 只。
同时买入全市场中得分最高且当前未持有的 k 只。
```

PDF 推荐：

```text
n = 5-30
k = 1-5
```

现实约束第一版可以简化，但报告中应说明是否考虑了：

- 手续费。
- 滑点。
- 涨跌停。
- 停牌。
- T+1。
- 最小交易单位。

## 10. 总结

这个项目的主线可以概括为：

```text
金融数据理解
-> 按交易日整理多股票面板
-> 构造只使用历史信息的特征
-> 生成未来收益标签
-> 用深度学习模型输出收益分数
-> 用分数排序交易
-> 用 IC/ICIR/回测验证
```

后续真正要把握住三件事：

1. 模型输入不是“当天 csv 文件”，而是“某只股票过去 L 个交易日的特征序列”。
2. 模型输出最好先理解为“未来收益排序分数”，不必执着于精确预测涨几个百分点。
3. 金融深度学习里，数据处理规范比模型花哨更重要。只要样本构造、时间切分、标准化和标签定义是干净的，即使模型很朴素，报告质量也会很扎实。
