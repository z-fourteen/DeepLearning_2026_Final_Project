# OXX 分支代码分析与工作流说明

> 面向创业板（ChiNext, 399006.SZ）的量化选股流水线：特征工程 → 序列模型 → 5日持有回测 → 组合优化

---

## 一、核心数据流

```
原始CSV数据
  → Ingest(Parquet)
    → State(交易状态：涨跌停/停牌/ST)
      → Pool(成分股SCD2)
        → Mart(特征面板 + 5日标签)
          → Clean Dataset(NPZ序列)
            → 模型训练(GRU/Transformer)
              → T+1回测模拟(5日持有)
                → LP组合优化器
                  → 组合权重输出
```

## 三、特征工程（两套特征集）

### 3.1 特征角色定义（`configs/features/advanced_sequence_clean_v1.yaml`）

所有特征分为4个角色，其中只有前两类进入模型：

| 角色 | 数量 | 是否进模型 | 说明 |
|------|------|-----------|------|
| `alpha_features` | **13个** | 是 | 核心预测特征 |
| `residualized_style_features` | **5个** | 是（新模式） | 风格中性化残差特征 |
| `risk_controls` | 12个 | 否 | 诊断用 |
| `tradability_controls` | 20个 | 否 | 诊断用 |

### 3.2 13个 Alpha 特征

```
lag1_net_mf_strength_20d_mean     -- 中期资金流强度
lag1_net_mf_strength_60d_mean     -- 长期资金流强度
lag1_close_position               -- 收盘价在20日高低范围中的位置
lag1_excess_ret_10d_mean          -- 中期超额收益
lag1_excess_ret_1d               -- 日度超额收益
lag1_excess_ret_5d_mean           -- 短期超额收益
lag1_industry_neutral_ret_1d      -- 行业中性收益
lag1_ret_1d                        -- 日度原始收益
lag1_ret_20d                       -- 20日收益
lag1_ret_5d_mean                   -- 5日均线收益
lag1_bollinger_z_20d              -- 布林带标准化
lag1_ma_ratio_20_60               -- 短/长期均线比率
lag1_macd_hist                     -- MACD 柱状图
```

### 3.3 "旧" vs "新" 特征集

通过 `--build-mode` 参数切换：

- **`alpha_only`（旧模式）：** 仅13个 alpha 特征，模型输入维度 = **13**
- **`alpha_plus_residual_style`（新模式）：** 13个 alpha + 5个残差风格特征，模型输入维度 = **18**

### 3.4 残差化处理

5个候选风格特征（`turnover_cost_proxy`, `turnover_20d_std`, `turnover_60d_std`, `amount_rank_pct`, `amount_log`）经过**横截面 OLS 回归中性化**：
- 设计矩阵 = 行业哑变量 + 标准化风格暴露（size/liquidity/volatility/momentum）
- 每个交易日独立回归，取残差作为最终特征

## 四、模型架构

### 4.1 通用组件（`src/models/base.py`）

**`FeatureProjection`** — 线性投影层
```
[B, T, F] → Linear(F, d_model) → LayerNorm → Dropout → [B, T, d_model]
```

**`PredictionHead`** — 两层 MLP 预测头
```
[B, H] → Linear(H, hidden_dim) → Activation → Dropout → Linear(hidden_dim, 1) → [B]
```

### 4.2 GRU 基线（`src/models/gru_model.py`）

```
Input [B, T, F]
  → FeatureProjection → [B, T, 64]
  → GRU(2层, hidden=128, dropout=0.2) → [B, T, 128]
  → 取最后隐藏状态 h_n[-1] → [B, 128]
  → LayerNorm → PredictionHead → [B]
```

- 适用训练脚本：`train_sequence.py`（`--model gru_baseline`）
- 支持 alpha_only（13维）和 alpha_plus_residual_style（18维）

### 4.3 Transformer 编码器（`src/models/transformer.py`）★ 来自 YXR

```
Input [B, T, F]
  → FeatureProjection → [B, T, 64]
  → Sinusoidal/Learnable 位置编码 → [B, T, 64]
  → (+CLS token if pooling=cls) → [B, T+1, 64]
  → TransformerEncoder(2层, 4头, FFN=128, Pre-LN) → [B, T+1, 64]
  → Pooling (cls / last_step / attention) → [B, 64]
  → LayerNorm → PredictionHead(GELU) → [B]
```

**三种池化方式：**
- `cls` — CLS token 直接输出（默认）
- `last_step` — 取最后时间步
- `attention` — `AttentionPooling`：`Linear→Tanh→Linear→Softmax→WeightedSum`

**关键配置：**
```yaml
model:
  name: transformer_encoder
  num_features: 18              # 或 13
  d_model: 64
  num_encoder_layers: 2
  num_heads: 4
  dim_feedforward: 128
  activation: gelu
  norm_first: true              # Pre-LN（重要）
  positional_encoding: sinusoidal  # 或 learnable
  pooling: cls                  # 或 last_step / attention
training:
  scheduler: warmup_cosine      # 10% 线性预热 + 余弦衰减
  learning_rate: 0.0003
```

- 适用训练脚本：`train_sequence_transformer.py`

### 4.4 FiLM-GRU（`src/models/feature_style_interaction_gru.py`）

```
Input [B, T, 18] → 拆分：
  alpha = x[..., :13]  → FeatureProjection → GRU → LayerNorm → alpha_context [B, 128]
  style = x[..., 13:]  → 取最后一天 → MLP(GELU) → style_context [B, 64]

FiLM 交互：
  style_context → Linear → gamma, beta
  gamma = 0.5 * tanh(gamma_raw)     # 缩放因子 ∈ [-0.5, 0.5]
  beta  = 0.2 * tanh(beta_raw)     # 偏移因子 ∈ [-0.2, 0.2]
  interacted = LayerNorm(alpha_context * (1 + gamma) + beta)

Output = head(interacted) + 0.25 * style_residual_head(style_context)
```

### 4.5 Regime-Gated GRU（`src/models/regime_gated_gru.py`）

```
Input [B, T, 18] → 拆分：
  alpha = x[..., :13]  → FeatureProjection → GRU → alpha_context [B, 128]
  style = x[..., 13:]  → FeatureProjection → GRU → style_context [B, 64]

Regime 检测：
  style最新日 batch统计 → Softmax → 3个市场状态权重
  3个专家头 → 加权组合 → alpha_score

门控：
  alpha_context + style_context → gate → [pos_scale, neg_scale, style_scale]
  pos_scale ∈ [0.5, 1.8]，neg_scale ∈ [0.2, 1.2]，style_scale ≤ 0.35

Output = pos_scale·ReLU(alpha_score) - neg_scale·ReLU(-alpha_score) + style_scale·style_adjust
```

## 五、训练框架

### 5.1 Trainer（`src/training/trainer.py`）

核心训练循环，支持：
- **早停：** 基于 `rank_ic_mean`，patience=10
- **坍塌检测：** 连续2个epoch出现 `prediction_collapse` 时立即停止
- **检查点策略：** 仅在 `daily_coverage_ratio >= 0.8` 时保存 best model
- **梯度裁剪：** `max_grad_norm = 1.0`
- **tqdm 进度条：** 训练/验证 batch 级别 + epoch 级别实时指标显示

### 5.2 损失函数（`src/training/losses.py`）

| 损失函数 | 公式 | 适用场景 |
|---------|------|---------|
| `HuberLoss` | PyTorch 内置 | Transformer 默认 |
| `MSEICLoss` | `(1-α)·MSE - α·IC` | GRU Alpha-Only（α=0.2） |
| `PearsonICLoss` | `-corr(pred, target)` | 纯 IC 优化 |
| `TopKMarginICLoss` | `ranking_loss + 0.2·IC + 0.02·MSE` | Top-K 排序优化 |

### 5.3 评估指标（`src/training/metrics.py`）

- **IC (Pearson IC)**：每日预测值与真实收益的相关系数
- **Rank IC (Spearman)**：排名相关性，对异常值更鲁棒
- **ICIR**：IC 的信息比率 = `mean(IC) / std(IC)`
- **诊断状态**：`prediction_collapse`（预测值方差极低）、`target_collapse`、`no_valid_daily_cross_section`

## 六、5日标签与回测

### 6.1 标签定义（`configs/data/labels.yaml`）

```yaml
default_horizon: 5                    # 5天持有期
label_mode: relative_return
benchmark: "399006.SZ"                 # 创业板指数
```

- `future_return = close[t+5] / close[t] - 1`（5日原始收益）
- `label_rel_return = future_return - benchmark_future_return`（5日超额收益）

### 6.2 时序分割（Purged Walk-Forward）

```
Train:      2016-01-04 ~ 2022-12-31
Purge:      2023-01-01 ~ 2023-01-31    (purge_days=5, 防止标签泄漏)
Validation: 2023-02-01 ~ 2024-12-31
Purge:      2025-01-01 ~ 2025-01-31
Test:       2025-02-01 ~ 2026-05-25
```

### 6.3 T+1 回测（`scripts/backtest/backtest_t1_fill_sim.py`）

**核心机制：**
1. **调仓频率：** 每5个交易日（`rebalance_stride`）
2. **选股：** 按 `pred_score` 降序排列 → Top-K
3. **排名缓冲（`keep_multiplier`）：** 先保留与当前持仓重叠的股票，再补充新股，减少换手
4. **T+1 模拟成交：**
   - 信号日 T 产生预测 → T+1 开盘价执行
   - `participation_cap = 0.03`（最大成交量为当日成交额的 3%）
   - 部分成交处理
5. **持有5日：** 从买入 T+1 开盘到卖出日收盘
6. **交易成本：** `10bps 佣金 + 5bps 滑点 = 15bps`

**输出：**
- `t1_fill_periods.csv` — 每期持仓明细
- `t1_fill_metrics.json` — 年化收益、IR、最大回撤、超额收益等

## 七、组合优化器（`scripts/portfolio/optimize_feasible_cash_buffer.py`）

使用 **cvxpy** 进行凸优化组合构建：

**目标函数（最大化）：**
```
pred_score^T · w
  - style_penalty · ||风格暴露||²
  - turnover_penalty · ||w - w_prev||₁
  - exposure_slack_penalty · Σ softplus(exposure - cap)
  - buy_capacity_slack_penalty · Σ softplus(buy_fill - capacity)
  - cash_penalty · (1 - Σw)
  - min_invested_shortfall_penalty · ReLU(0.80 - Σw)
```

**硬约束：**
- `0 ≤ wᵢ ≤ 0.10`（单票上限 10%）
- `Σw ≥ 0.80`（最低持仓 80%）
- 成交量约束、T+1 可执行性过滤

**风险控制集：**

| 名称 | 暴露因子数 | 说明 |
|------|-----------|------|
| `none` | 0 | 无风格约束 |
| `industry_proxy` | 4 | 行业排名 |
| `industry_size` | 6 | 行业 + 市值 |
| `industry_size_liquidity_vol_mom` | 16 | 全风格约束 |

## 八、完整执行工作流

### Stage 1：数据摄取
```bash
python scripts/run_daily_dag.py                    # 一键执行
python scripts/run_daily_dag.py --skip-ingest-raw   # 跳过已有步骤
```
执行顺序：`ingest_raw → build_pool → build_market_state → validate_state → build_mart`

### Stage 2：构建 Clean Dataset
```bash
python scripts/modeling/build_clean_model_datasets.py \
  --build-mode alpha_only              # 旧特征集（13维）
# 或
python scripts/modeling/build_clean_model_datasets.py \
  --build-mode alpha_plus_residual_style  # 新特征集（18维）
```
输出 NPZ 文件到 `data/clean/` 目录。

### Stage 3：模型训练
```bash
# GRU 系列（旧脚本）
python scripts/modeling/train_sequence.py \
  --config configs/sequence_gru.yaml

# Transformer（新脚本）
python scripts/modeling/train_sequence_transformer.py \
  --config configs/sequence_transformer_attn_pool.yaml
```

### Stage 4：回测
```bash
python scripts/backtest/backtest_t1_fill_sim.py \
  --predictions outputs/runs/xxx/predictions.parquet
```

### Stage 5：组合优化
```bash
python scripts/portfolio/optimize_feasible_cash_buffer.py
```
